# Codex依頼プロンプト：SlideRoom UI完成後の機能実装

以下の仕様に従って、すでにUIが完成しているSlideRoomへ機能実装を追加してください。

## 前提

SlideRoomは、大学・ゼミ・グループワーク向けのPowerPoint提出・管理・結合Webアプリです。  
すでにホーム、ルーム、提出ファイル、スライド順番、結合、設定のUIはモック画像をもとに完成している前提です。

今回の目的は、UIを大きく変更せず、**実際に操作できるMVPアプリ**にすることです。

## 参照する設計書

- `SlideRoom_UI完成後_機能実装設計書.md`
- 既存の `SlideRoom_開発設計書.md`
- 既存のUIモック画像一式

## 絶対に守ること

1. 既存UIの見た目を大きく変えないでください。
2. UIを壊さず、状態管理・保存処理・操作処理を追加してください。
3. 本物の既存PPTX完全結合はMVPでは不要です。
4. ただし結合処理は `mergeService` に分離し、将来本格エンジンへ差し替え可能にしてください。
5. PCとスマホのレスポンシブ表示を維持してください。
6. TypeScriptの型を丁寧に定義してください。
7. モックデータだけでなく、ユーザー操作で追加・変更したデータが画面に反映されるようにしてください。
8. データは再読み込み後も残るように保存してください。

## 追加してほしいライブラリ

必要に応じて以下を追加してください。

```bash
npm install zustand idb nanoid date-fns sonner
npm install pptxgenjs jszip file-saver
```

既存プロジェクトで別の状態管理やトーストが入っている場合は、それを使っても構いません。

### 3. 永続化

- ルーム、メンバー、ファイルメタデータ、スライド順番、設定を保存してください。
- 実ファイルBlobは可能ならIndexedDBへ保存してください。
- まずはlocalStorage中心でも構いませんが、ファイルBlob保存はサービス層に分離してください。
- アプリ再読み込み後も、作成したルームや並び替えたスライド順が保持されるようにしてください。

### 4. ルーム作成・参加

実装すること。

- ホームの「ルームを作成」ボタンから作成モーダルを開く
- ルーム名、授業名、チーム名、発表日時、提出期限を入力できる
- 作成後、ルーム一覧とホームに反映する
- 自分をホストとしてメンバー追加する
- 招待コードを生成する
- 「ルームに参加」から招待コードで参加できる
- 存在しないコードの場合はエラー表示
- 招待URLコピーでクリップボードへコピーし、トースト表示

### 5. ルーム一覧

実装すること。

- 検索
- ステータスフィルタ
- 更新日時または提出期限で並び替え
- ルームカードクリックで詳細へ遷移
- 参加中、提出待ち、結合準備OKなどの数値を自動計算

### 6. ルーム詳細

ルーム詳細内で以下を実装してください。

- 概要タブ
- メンバータブ
- 提出ファイルタブ
- スライド順番タブ
- 設定タブ

概要タブでは以下を表示してください。

- 進捗率
- 提出済み人数
- 未提出人数
- 合計メンバー数
- ルーム情報
- やること
- 最近の更新
- スライド構成

### 7. ファイルアップロード

提出ファイルタブで以下を実装してください。

- ファイル選択アップロード
- ドラッグ&ドロップアップロード
- 複数ファイルアップロード
- 許可形式チェック
- 最大サイズチェック
- ファイルメタデータ作成
- 提出者名・ルーム名・更新日時・ステータス表示
- ファイル削除
- ファイル差し替え
- ステータス変更

対応形式。

- pptx
- ppt
- pdf
- docx
- xlsx

MVPでは本物のPowerPointプレビューは不要です。ファイル情報とダミーサムネイルで構いません。

### 8. スライドデータ生成

ファイルがアップロードされたら、そのファイルから仮の `SlideItem` を生成してください。

- PPTXの場合は `slideCount` を仮に1〜5枚で生成してよい
- PDFの場合も仮生成でよい
- 既存モックに近いタイトルを推定してよい
- SlideItemには fileId、ownerName、title、sourcePage、order、section を持たせる

### 9. スライド順番画面

実装すること。

- `@dnd-kit/sortable` などでドラッグ&ドロップ並び替え
- 並び替え後にstoreへ保存
- 「順番を保存」ボタンで保存トースト表示
- ルームごとの並び順を再読み込み後も保持
- 合計スライド数、未配置、重複のチェック表示
- スマホではカード型リストで操作できるようにする

### 10. 結合画面

実装すること。

- 選択中または最新のルーム情報を表示
- 合計スライド数、メンバー数、結合準備状況を表示
- 結合チェックを実データから計算
- スライド順のプレビューを表示
- 出力設定を変更できる
- ファイル名を編集できる
- PPTX出力ボタンを動かす
- PDF出力ボタンは準備中トーストでもよい
- 出力履歴を追加する

### 11. PPTX出力

MVPとして、PptxGenJSを使って簡易PPTXを生成してください。

内容。

- 表紙スライド
- スライド順番に従った簡易ページ
- 各ページに以下を表示
  - スライド番号
  - スライドタイトル
  - 担当者
  - 元ファイル名
  - セクション名
- 最終ページに「SlideRoomで出力」などの表記

注意。

これは既存PPTXの完全結合ではなく、MVPの操作確認用です。  
`mergeService` の中に実装し、将来本格結合エンジンへ差し替えられるようにしてください。
